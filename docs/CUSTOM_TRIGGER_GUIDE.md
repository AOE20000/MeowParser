# 自定义触发器和直接输入模式指南

## 概述

MeowParser v2.2.1 新增了窗口级别的高级配置功能：
- **自定义触发器**：为不同窗口设置不同的触发键
- **直接输入模式**：跳过点击流程，直接输入文本

---

## 自定义触发器

### 什么是自定义触发器？

默认情况下，MeowParser 在任何输入后按空格键触发悬浮窗。自定义触发器允许你为特定窗口设置特殊的触发键组合。

### 使用场景

**场景 1：避免误触发**
- 在某些应用中，你可能经常输入空格
- 设置触发器为 `u`，只有输入 `u` 后按空格才触发

**场景 2：快速触发**
- 设置简短的触发键，如 `m`
- 输入 `m` + 空格即可快速触发

### 配置方法

1. 打开窗口管理器（托盘图标 → 窗口管理）
2. 找到目标窗口
3. 右键点击窗口 → 选择"⚙️ 配置窗口..."
4. 在"触发键"输入框中输入触发字符（例如：`u`）
5. 点击"确定"保存

### 示例

**设置触发器为 "u"：**
```
正常输入：你好世界
触发悬浮窗：u[空格]
```

当你输入 `u` 后按空格，会触发悬浮窗，且 `u` 会被自动移除。

**留空使用默认：**
```
触发悬浮窗：任何输入 + [空格]
```

## 直接输入模式

### 什么是直接输入模式？

标准模式下，MeowParser 的工作流程：
1. 触发悬浮窗
2. 输入文本并回车
3. 点击原输入框位置
4. 输入处理后的文本
5. 按回车发送

直接输入模式跳过第3步（点击），直接输入文本并回车。

### 使用场景

**适用窗口：**
- 聊天应用（QQ、微信、Discord等）
- 命令行终端
- 任何"始终保持输入焦点"的窗口

**不适用窗口：**
- 需要点击输入框才能输入的网页
- 需要手动选择输入框的应用

### 配置方法

1. 打开窗口管理器（托盘图标 → 窗口管理）
2. 找到目标窗口
3. 右键点击窗口 → 选择"⚙️ 配置窗口..."
4. 勾选"直接输入模式"
5. 点击"确定"保存

### 工作流程对比

**标准模式：**
```
1. 触发悬浮窗
2. 输入文本
3. 回车
4. → 点击原位置
5. → 输入文本
6. → 按回车
```

**直接输入模式：**
```
1. 触发悬浮窗
2. 输入文本
3. 回车
4. → 恢复窗口焦点
5. → 直接输入文本
6. → 按回车
```

## 窗口配置界面

### 访问方式

**方法 1：右键菜单**
1. 打开窗口管理器
2. 右键点击窗口
3. 选择"⚙️ 配置窗口..."

**方法 2：双击切换**
- 双击窗口可快速启用/禁用
- 需要详细配置时使用右键菜单

### 配置选项

#### 基本设置
- **启用此窗口**：是否在此窗口中启用 MeowParser

#### 触发器设置
- **触发键**：自定义触发字符（留空使用默认）
- 示例：设置为 `u`，则输入 `u` 后按空格触发

#### 输入模式
- **直接输入模式**：启用后跳过点击流程
- 适用于始终保持输入焦点的窗口

## 窗口管理器功能

### 列表显示

窗口列表显示以下信息：
- **进程名**：应用程序名称
- **窗口标题**：窗口的标题文本
- **状态**：✓ 已启用 / ✗ 未启用

### 快捷操作

- **双击**：快速启用/禁用窗口
- **右键**：打开配置菜单
  - ⚙️ 配置窗口...
  - ✅ 启用窗口 / ❌ 禁用窗口
- **搜索**：在搜索框中输入关键词过滤窗口
- **排序**：点击列标题排序

## 配置文件格式

窗口配置保存在 `.meowparser/window_settings.json`：

```json
{
  "应用名 - 窗口标题": {
    "enabled": true,
    "trigger_key": "u",
    "direct_input": true
  }
}
```

**字段说明：**
- `enabled`：是否启用（布尔值）
- `trigger_key`：触发键（字符串，空字符串表示默认）
- `direct_input`：是否使用直接输入模式（布尔值）

## 常见问题

### Q1：触发器不工作？

**检查：**
1. 确认窗口已启用
2. 确认触发键设置正确
3. 确认输入的是小写字母（触发器不区分大小写）

### Q2：直接输入模式下文本没有输入？

**原因：**
- 窗口失去了输入焦点
- 窗口需要点击才能输入

**解决：**
- 禁用直接输入模式
- 使用标准模式

### Q3：如何恢复默认设置？

**方法：**
1. 打开窗口管理器
2. 右键点击窗口 → 配置窗口
3. 清空触发键
4. 取消勾选直接输入模式
5. 点击确定

### Q4：可以为所有窗口设置相同的触发器吗？

**回答：**
- 目前每个窗口需要单独配置
- 未来版本可能添加全局默认设置

## 技术细节

### 触发器匹配逻辑

```python
# 检查输入缓冲区是否以触发键结尾
if input_buffer.endswith(trigger_key):
    # 触发悬浮窗
    # 移除触发键
    input_buffer = input_buffer[:-len(trigger_key)]
```

### 直接输入模式流程

```python
if direct_input_mode:
    # 只恢复窗口焦点
    restore_window_focus()
    # 直接输入文本
    keyboard.write(text)
    keyboard.press('enter')
else:
    # 标准模式：恢复焦点 + 点击 + 输入
    restore_window_focus()
    click_at_position()
    keyboard.write(text)
    keyboard.press('enter')
```
