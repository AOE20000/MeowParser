## 🆕 新增功能

### 自定义触发器

**功能说明：**
- 默认：任何输入后按空格触发
- 自定义：设置特定字符（如 `u`），输入该字符后按空格触发

**使用场景：**
- 避免误触发：在经常输入空格的应用中
- 快速触发：设置简短的触发键

**配置方法：**
1. 打开窗口管理器
2. 右键点击窗口 → "⚙️ 配置窗口..."
3. 在"触发键"输入框中输入字符
4. 点击"确定"保存

### 直接输入模式

**功能说明：**
- 标准模式：点击输入框 → 输入文本 → 回车
- 直接模式：直接输入文本 → 回车（跳过点击）

**适用场景：**
- 聊天应用（QQ、微信、Discord）
- 命令行终端
- 任何始终保持输入焦点的窗口

**配置方法：**
1. 打开窗口管理器
2. 右键点击窗口 → "⚙️ 配置窗口..."
3. 勾选"直接输入模式"
4. 点击"确定"保存

### 规则组管理增强

**新增功能：**
- 上移规则组：调整规则组执行顺序
- 下移规则组：调整规则组执行顺序
- 正则标识：橙色高亮显示正则表达式规则

**操作方式：**
- 右键规则组 → 选择"上移规则组"或"下移规则组"
- 规则列表中，正则规则显示 "✓ 正则"（橙色）

## 📁 配置文件变更

### 新的配置位置

```
.meowparser/
├── rules/                    # 规则配置
│   └── *.json
├── window_settings.json      # 窗口设置（新位置）
└── instance.lock             # 实例锁（Linux）
```

### 自动迁移

程序首次运行时会自动：
1. 检测旧配置文件（`window_settings.json` 在根目录）
2. 迁移到新位置（`.meowparser/window_settings.json`）
3. 删除旧文件

### 配置文件格式

**窗口配置：**
```json
{
  "应用名 - 窗口标题": {
    "enabled": true,
    "trigger_key": "u",
    "direct_input": true
  }
}
```

## 🎯 使用建议

### 推荐配置

**QQ/微信聊天窗口：**
- 触发键：`u`
- 直接输入：✓ 启用

**浏览器网页：**
- 触发键：（默认）
- 直接输入：✗ 禁用

**命令行终端：**
- 触发键：`m`
- 直接输入：✓ 启用

### 最佳实践

1. **触发器设置**
   - 选择不常用的字符作为触发键
   - 避免使用常见字母（如 a、e、i）

2. **直接输入模式**
   - 只在始终保持焦点的窗口中启用
   - 测试后再决定是否启用

3. **规则组管理**
   - 按功能分类创建规则组
   - 调整规则组顺序优化执行效率
   - 使用正则表达式提高规则灵活性
