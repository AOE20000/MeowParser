# UI 改进说明

## 配置编辑器界面优化

### v2.2.1 改进

#### 规则列表显示优化

**改进前：**
```
列标题：[规则] [操作]
规则显示：
  ☑ 标点前添加喵    编辑 | 删除 | ↑ | ↓
  ☑ 换行前添加喵    编辑 | 删除 | ↑ | ↓
```

**改进后：**
```
列标题：[规则] [正则]
规则显示：
  ☑ 标点前添加喵    ✓ 正则  (橙色高亮)
  ☑ 换行前添加喵    ✓ 正则  (橙色高亮)
  ☑ 自称为喵              (普通文本规则)
```

---

## 改进详情

### 1. 列标题优化

**旧设计问题：**
- "操作"列显示 "编辑 | 删除 | ↑ | ↓"
- 信息冗余，占用空间
- 用户可以通过右键菜单或双击操作，不需要文字提示

**新设计优势：**
- "正则"列直接显示规则类型
- 一目了然识别正则表达式规则
- 节省空间，信息更集中

### 2. 正则表达式标识

**视觉标识：**
- 正则规则：显示 "✓ 正则"
- 普通规则：留空

**颜色编码：**
- 正则规则：橙色 (#FFA500)
- 规则文本：浅灰色 (#E0E0E0)
- 普通规则：默认颜色

### 3. 规则组显示

**规则组行：**
- 第一列：显示规则组名称和统计
- 第二列：留空（不显示正则标识）

**示例：**
```
▼ 喵语转换 (7/8条规则)
  ☑ 句号、感叹号、问号前添加喵    ✓ 正则
  ☑ 逗号、顿号、分号前添加喵      ✓ 正则
  ☑ 冒号前添加喵                  ✓ 正则
  ☑ 右括号前添加喵                ✓ 正则
  ☑ 换行前添加喵                  ✓ 正则
  ☑ 句尾添加喵                    ✓ 正则
  ☐ 自称为喵
```

## 用户体验提升

### 快速识别规则类型

**场景 1：查看配置**
- 用户打开配置编辑器
- 一眼看出哪些是正则规则
- 橙色标识醒目易识别

**场景 2：调试规则**
- 某个规则不工作
- 快速确认是否使用正则
- 检查正则语法是否正确

**场景 3：学习参考**
- 新用户查看默认配置
- 了解哪些规则使用正则
- 学习正则表达式用法

### 操作便捷性

**保留的操作方式：**
1. **双击规则**：切换启用/禁用
2. **双击规则组**：展开/折叠
3. **右键菜单**：
   - 编辑规则
   - 删除规则/规则组
   - 上移/下移
   - 添加规则

**移除的冗余信息：**
- 不再在列中显示 "编辑 | 删除 | ↑ | ↓"
- 用户通过右键菜单访问这些功能
- 界面更简洁清爽

## 技术实现

### 颜色设置

```python
from PyQt6.QtGui import QBrush, QColor

# 正则规则标识
if is_regex:
    rule_item.setText(1, "✓ 正则")
    # 橙色高亮
    rule_item.setForeground(1, QBrush(QColor("#FFA500")))
    # 规则文本浅灰色
    rule_item.setForeground(0, QBrush(QColor("#E0E0E0")))
else:
    rule_item.setText(1, "")
```

### 列宽设置

```python
self.tree.setHeaderLabels(['规则', '正则'])
self.tree.setColumnWidth(0, 650)  # 规则列更宽
self.tree.setColumnWidth(1, 80)   # 正则列固定宽度
```

## 视觉设计原则

### 1. 信息层次

**主要信息：**
- 规则描述（大字体，主要位置）
- 启用状态（☑/☐）

**次要信息：**
- 正则标识（小字体，右侧）
- 规则统计（括号内）

### 2. 颜色使用

**功能性颜色：**
- 橙色：正则表达式（警示/特殊）
- 绿色：已启用（✓）
- 红色：未启用（✗）
- 灰色：次要信息

**避免过度使用：**
- 不是所有规则都高亮
- 只标识特殊类型（正则）
- 保持界面整洁

### 3. 空间利用

**优化前：**
- "操作"列占用 ~150px
- 显示冗余文字信息

**优化后：**
- "正则"列只需 80px
- 规则列增加 70px 宽度
- 显示更多规则内容
