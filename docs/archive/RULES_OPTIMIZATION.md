# 喵语规则优化说明

## 优化版本：v2.0

### 优化目标

使喵语转换规则能够智能处理各种场景，包括：
- 保护 URL 和配置项（如 `about:config`、`privacy.exposeContentTitleInWindow`）
- 正确处理中英文混合文本
- 支持无标点符号的换行
- 避免破坏技术内容的完整性

---

## 规则说明

### 1. 句号、感叹号、问号前添加"喵"
- **模式**: `([^喵\s。，！？；、）（：])([。！？])`
- **示例**: 
  - `你好。` → `你好喵。`
  - `为什么？` → `为什么喵？`
  - `太棒了！` → `太棒了喵！`

### 2. 逗号、顿号、分号前添加"喵"
- **模式**: `([^喵\s。，！？；、）（：])([，、；])`
- **示例**:
  - `你好，世界` → `你好喵，世界喵`
  - `苹果、香蕉、橙子` → `苹果喵、香蕉喵、橙子喵`

### 3. 冒号前添加"喵"（智能保护）
- **模式**: `([^喵\s。，！？；、）（：])([：])(?![a-zA-Z0-9_\-\.:/])`
- **保护规则**: 冒号后如果紧跟字母、数字、下划线、连字符、点、斜杠等，则不添加"喵"
- **示例**:
  - `注意：这很重要` → `注意喵：这很重要喵` ✓
  - `输入：privacy.test` → `输入：privacy.test喵` ✓（保护配置项）
  - `访问about:config` → `访问about:config喵` ✓（保护URL）

### 4. 右括号前添加"喵"
- **模式**: `([^喵\s。，！？；、）（：])(）)`
- **示例**:
  - `（没有问题）继续` → `（没有问题喵）继续喵`
  - `这是（一个（嵌套）的）例子` → `这是（一个（嵌套喵）的喵）例子喵`

### 5. 换行前添加"喵"
- **模式**: `([^喵。，！？；、）（：\n])(\n)`
- **重要**: 支持无标点符号的换行
- **示例**:
  - `第一行\n第二行` → `第一行喵\n第二行喵`
  - `这是一行\n这是另一行` → `这是一行喵\n这是另一行喵` ✓

### 6. 句尾添加"喵"
- **模式**: `([^喵。，！？；、）（：\n\s.])$`
- **保护规则**: 排除英文句号结尾
- **示例**:
  - `你好世界` → `你好世界喵`
  - `Hello world.` → `Hello world.` ✓（保护英文句子）

## 智能保护特性

### URL 和配置项保护
规则会自动识别并保护以下内容：
- URL: `about:config`, `https://example.com`
- 配置项: `privacy.exposeContentTitleInWindow`
- 文件路径: `C:\Users\test.txt`
- 邮箱地址: `test@example.com`
- 版本号: `3.14.159`

### 避免重复添加
- 已有"喵"的位置不会重复添加
- 示例: `你好喵。` → `你好喵。` ✓

### 标点符号智能处理
- 连续标点只在第一个前添加"喵"
- 示例: `什么？！` → `什么喵？！`

## 技术细节

### 正则表达式说明

- `[^喵\s。，！？；、）（：]`: 排除已有"喵"、空格和标点符号
- `(?![a-zA-Z0-9_\-\.:/])`: 负向前瞻，确保后面不是URL或配置项的一部分
- `\n`: 匹配换行符
- `$`: 匹配字符串结尾

### 规则执行顺序

规则按顺序执行，确保：
1. 先处理标点符号前的"喵"
2. 再处理换行前的"喵"
3. 最后处理句尾的"喵"
