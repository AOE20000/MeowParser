# 规则组使用指南

## 什么是规则组？

规则组是 MeowParser 2.1.0 引入的新功能，允许你创建多个独立的替换规则集合，并可以根据不同场景灵活启用或禁用。

---

## 为什么需要规则组？

### 使用场景

1. **不同聊天场景**
   - 工作群组：使用正式的替换规则
   - 朋友群组：使用轻松的替换规则
   - 游戏频道：使用游戏相关的替换规则

2. **不同语言环境**
   - 中文规则组
   - 英文规则组
   - 混合规则组

3. **测试和生产**
   - 测试规则组：用于测试新规则
   - 生产规则组：稳定的日常使用规则

---

## 如何使用规则组？

### 创建规则组

1. 右键托盘图标 → 替换规则
2. 点击左侧"新建组"按钮
3. 输入规则组名称（如"工作场景"）
4. 点击确定

### 管理规则组

- **重命名**：选择规则组 → 点击"重命名"按钮
- **删除**：选择规则组 → 点击"删除组"按钮
- **启用/禁用**：双击规则组名称即可切换状态

### 添加规则

1. 选择一个规则组
2. 点击右侧"添加规则"按钮
3. 输入匹配文本和替换文本
4. 选择是否使用正则表达式
5. 点击确定

### 调整规则顺序

- 选择规则 → 点击"上移"或"下移"按钮
- 规则按从上到下的顺序依次应用

## 配置示例

### 示例1：工作和生活分离

```json
{
  "groups": {
    "工作场景": {
      "enabled": true,
      "rules": [
        {"pattern": "好的", "replacement": "收到", "is_regex": false},
        {"pattern": "谢谢", "replacement": "感谢", "is_regex": false}
      ]
    },
    "朋友聊天": {
      "enabled": false,
      "rules": [
        {"pattern": "我", "replacement": "喵", "is_regex": false},
        {"pattern": "你", "replacement": "喵", "is_regex": false}
      ]
    }
  }
}
```

### 示例2：多语言支持

```json
{
  "groups": {
    "中文规则": {
      "enabled": true,
      "rules": [
        {"pattern": "我", "replacement": "喵", "is_regex": false}
      ]
    },
    "英文规则": {
      "enabled": true,
      "rules": [
        {"pattern": "\\bI\\b", "replacement": "meow", "is_regex": true}
      ]
    }
  }
}
```

### 示例3：正则表达式高级用法

```json
{
  "groups": {
    "数字处理": {
      "enabled": true,
      "rules": [
        {"pattern": "\\d+", "replacement": "数字", "is_regex": true}
      ]
    },
    "表情符号": {
      "enabled": false,
      "rules": [
        {"pattern": "😊", "replacement": "喵~", "is_regex": false}
      ]
    }
  }
}
```

## 规则执行顺序

1. 按规则组在配置文件中的顺序
2. 只应用启用状态的规则组
3. 在每个规则组内，按规则的顺序依次应用
4. 所有处理都通过规则组完成（包括"喵"字添加）

### 示例

假设有以下配置：

```
规则组A（启用）：
  - "我" → "喵"
  
规则组B（禁用）：
  - "你" → "喵"
  
喵语转换（启用）：
  - 在标点前添加"喵"
  - 在句尾添加"喵"
```

输入："我和你不知道。"

执行过程：
1. 应用规则组A："喵和你不知道。"
2. 跳过规则组B（禁用）
3. 应用喵语转换："喵和你不知道喵。"

## 内置规则组说明

### 喵语转换规则组

这是一个特殊的内置规则组，用于添加"喵"字：

```json
{
  "喵语转换": {
    "enabled": true,
    "rules": [
      {
        "pattern": "([^喵])([。，！？；、）（：])",
        "replacement": "\\1喵\\2",
        "is_regex": true
      },
      {
        "pattern": "([^喵。，！？；、）（：\\n])(\\n)",
        "replacement": "\\1喵\\2",
        "is_regex": true
      },
      {
        "pattern": "([^。，！？；、）（：\\n])$",
        "replacement": "\\1喵",
        "is_regex": true
      }
    ]
  }
}
```

**功能**：
1. 在标点符号前添加"喵"（如果前面不是"喵"）
2. 在换行符前添加"喵"（如果前面不是标点或"喵"）
3. 在句尾添加"喵"（如果末尾不是标点或换行）

**自定义**：
- 可以禁用此规则组来关闭"喵"字添加
- 可以修改规则来自定义添加逻辑
- 可以创建自己的文本处理规则组

## 最佳实践

### 1. 合理命名

使用清晰的规则组名称：
- ✅ "工作场景"、"朋友聊天"
- ❌ "规则1"、"测试"

### 2. 避免规则冲突

注意规则之间的相互影响：
```
错误示例：
规则1："我" → "你"
规则2："你" → "他"
结果："我" → "你" → "他"（可能不是预期）

正确示例：
使用不同的规则组分别管理
```

### 3. 测试新规则

- 创建"测试"规则组
- 在测试组中添加新规则
- 确认无误后再移到正式规则组

### 4. 定期清理

- 删除不再使用的规则组
- 合并相似的规则
- 保持配置文件简洁

## 常见问题

### Q: 规则组的顺序重要吗？

A: 是的。规则组按配置文件中的顺序依次应用。如果规则之间有依赖关系，需要注意顺序。

### Q: 可以同时启用多个规则组吗？

A: 可以。所有启用的规则组都会按顺序应用。

### Q: 如何临时禁用某个规则？

A: 双击规则组名称即可快速切换启用/禁用状态。

### Q: 如何关闭"喵"字添加功能？

A: 禁用"喵语转换"规则组即可。双击该规则组名称可快速切换。

### Q: 可以自定义"喵"字添加逻辑吗？

A: 可以。编辑"喵语转换"规则组中的正则表达式规则，或创建自己的规则组。

### Q: 如何分享我的规则组？

A: 选择规则组 → 点击"导出规则组"按钮 → 保存为 JSON 文件 → 分享给其他人。

### Q: 如何使用别人分享的规则组？

A: 点击"导入规则组"按钮 → 选择 JSON 文件 → 自动导入到规则列表。

### Q: 导入的规则组会覆盖现有规则吗？

A: 不会。导入的规则组会作为新的规则组添加，不会影响现有规则。

## 配置文件位置

- Windows: `.\replacement_rules.json`
- Linux/macOS: `./replacement_rules.json`

可以手动编辑此文件，但建议使用图形界面管理。

## 导入和导出规则组

### 导出规则组

1. 打开替换规则管理窗口
2. 选择要导出的规则组
3. 点击"导出规则组"按钮
4. 选择保存位置和文件名
5. 规则组将保存为 JSON 文件

**导出格式示例**：
```json
{
  "enabled": true,
  "rules": [
    {
      "pattern": "我",
      "replacement": "喵",
      "is_regex": false
    }
  ]
}
```

### 导入规则组

1. 打开替换规则管理窗口
2. 点击"导入规则组"按钮
3. 选择 JSON 文件
4. 规则组将自动添加到列表中

**支持的格式**：
- 单个规则组文件
- 多个规则组文件（包含 `groups` 字段）

**冲突处理**：
- 如果规则组名称已存在，会提示输入新名称
- 或自动重命名为 `规则组名_1`、`规则组名_2` 等

### 规则组分享

**分享你的规则组**：
1. 导出规则组为 JSON 文件
2. 分享文件给其他用户
3. 可以在社区或论坛发布

**使用他人的规则组**：
1. 下载规则组 JSON 文件
2. 使用"导入规则组"功能导入
3. 根据需要启用或修改

**人称替换规则组**：
```json
{
  "enabled": true,

    {"pattern": "你", "replacement": "喵", "is_regex": false},
    {"pattern": "他", "replacement": "喵", "is_regex": false}
  ]
}
```

**数字处理规则组**：
```json
{
  "enabled": true,
  "rules": [
    {"pattern": "\\d+", "replacement": "数字", "is_regex": true}
  ]
}
```
